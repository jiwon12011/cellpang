# 세포팡 기획서 (v1)

> "유미의 세포들" IP 모바일 매치-3 **힐링** 퍼즐 PWA.
> 방향: **1판 = 1장면 감정 서사 × 시네마틱 콤보 다이브.**
> 작성: 우리팀(team-lead·ideator·developer·designer·motion-engineer·perf-engineer) 협업.

---

## 0. 포지셔닝

> **"이기는 매치-3가 아니라, 오늘의 내 감정을 돌보는 매치-3."**

- 랭킹·하트 대기·후반 난이도 벽·점수 압박 = **배제**.
- 한 끗: *한 판이 끝나면 점수가 아니라 오늘 유미의 한 컷이 남는다.* 그리고 결정적 순간엔 화면이 세포 마을 속으로 빨려 들어간다.

---

## 1. 게임 루프

- 보드 **6열 × 7행 세로**, 상단 1/3 "세포 마을 무대" + 하단 2/3 보드.
- **타이머 없음 → 턴(이동 횟수) 기반.** 째깍거리는 압박 금지.
- 한 판(60~90초) 4단계:

```
[진입 컷]  정지 일러 + 말풍선 (탭 스킵) → 미션 카드 제시
   ↓
[보드 미션]  매치하며 목표 달성. 무대 세포 실시간 리액션. 콤보로 감정 게이지 충전.
   ↓
[분기]  목표 달성 = 클리어 / 턴 소진 = "그런 날"(소프트 실패, 벌점 없음)
   ↓
[엔딩 컷]  클리어 4컷 / 실패 2컷 슬라이드 → "다음 장면" 또는 "한 번 더"
```

## 2. 미션 (MVP 3종)

| 미션명 | 메커닉 | 매핑 감정 |
|--------|--------|-----------|
| **마음 채우기** | 특정 세포 N개 매치 | 설렘·애정 |
| **두근두근 연쇄** | 콤보/연쇄 N회 달성 | 고조되는 감정 |
| **먹구름 걷어내기** | 방해물(불안·먹구름 타일) 전부 제거 | 불안 해소 |

> 2차 확장: 출출세포에게 배달 / 마음 정리 / 잠깐 쉬어가기.

## 3. 챕터 1 "보통의 하루"

하루 = 1챕터, 시간대별 4장면. **실제 시간대와 동기화**(아침에 열면 아침 장면) → 억지 출석 없이 자연 복귀.

1. **아침 — "오늘도 출근"** · 먹구름 걷어내기
2. **점심 — "출출세포의 시간"** · 마음 채우기
3. **오후 — "그 사람 메시지"** · 두근두근 연쇄
4. **밤 — "오늘 하루 정리"** · 마음 정리(2차)

하루 4장면 완료 시 그날의 **"오늘의 마을 한 컷"** 1장(메인 세포 + 한 줄) → 향후 공유 카드의 씨앗.

## 4. 시네마틱 콤보 다이브

- **발동**: 감정 게이지(콤보/연쇄로만 충전)가 풀 차면 1회. **한 판 0~1회** (남발 금지).
- **연출(0.95초, WAAPI)**: 보드 살짝 축소 → `.dive-cam` 단일 노드로 무대 줌인(레이어 패럴랙스) → 세포 마을 풀샷 컷인(환호/하트/햇살) → 줌아웃 복귀.
- **게임 효과(결정)**: **방해물 + 목표 세포 일괄 정리** 단일 효과(미션별 분기 안 함 → engine 결합도 최소, "막힌 게 시원히 풀리는" 카타르시스).
- 게이지는 **숫자 비노출**, "감정이 차오른다" 내러티브 라벨로만.

## 5. 기술 아키텍처

**권고: 통째 재작성 ✕ → 점진 확장.** 현 `game.js` 코어(매치/중력/리필/캐스케이드/입력)는 검증됨 → 재사용.

```
index.html  → <script type="module" src="./src/main.js">
src/
  main.js     오케스트레이터 + 상태머신
  engine.js   [재사용] grid/findMatches/swap/resolveBoard/collapseAndRefill + 이벤트 발행
  input.js    [재사용] 드래그/탭 → engine.trySwap, 입력잠금 단일 게이트
  mission.js  미션 목표 판정 (engine 이벤트 구독)
  scene.js    컷 시퀀서 (정지 일러+말풍선, 탭 스킵)
  gauge.js    감정 게이지 충전 + 다이브 트리거
  state.js    localStorage 진행 저장/로드 + 시간대 라우터
data/chapter1.json   장면·미션 정의(코드 아닌 데이터)
```

핵심 원칙:
- **engine은 서사를 모른다.** mission/gauge/scene이 engine 이벤트(`onMatch`/`onCascade`/`onResolveEnd`/`onObstacleCleared`)를 **구독만**.
- **motion은 콜백 레지스트리**(`setMotionHooks`)로 주입 — engine은 GSAP을 import하지 않음. 각 hook은 Promise 반환 가능, `prefers-reduced-motion`은 빈 콜백.

**상태 머신**: `intro_cut → playing → resolved(clear/soft_fail) → ending_cut → next`. 입력잠금 = `phase==='playing' && !engine.isResolving`.

**코어 변경 범위**: `SIZE` → `ROWS/COLS`(6×7), 타이머 → `movesLeft`, `tile.kind`(normal/obstacle) 추가, **`hasPossibleMove()` + 자동 셔플 신규**(데드락 차단).

### 데이터 모델
- 장면/미션: `chapter1.json`(sceneId, timeBand, board{cols,rows}, moves, mission{type,target}, introCut, endingCut{clear,soft_fail}).
- 진행: `localStorage["yumi-cellpang-save"]` — `{ v, currentChapter, unlocked, cleared, lastVisit, todayScene, oneCutOfDay }`. 파싱 실패/버전 불일치 시 안전 리셋.

## 6. 비주얼 시스템

기준 393×852 PWA standalone. 세로 스택: **HUD 56 / 무대 236 / 보드 416** (px).
- 보드 6열 cell ≈ **52px** (≥ 44 터치 충족). HUD: 좌=미션 목표 · 중=남은 턴(기존 타이머바→턴 게이지) · 우=진행.
- **핵심 축: 시간대 무드는 body + 무대에만, 보드는 항상 흰 무대(명도 96%↑)로 고정** → 타일 6색 대비가 시간대와 무관하게 일정(가독성·일관성 보증). *밤 팔레트만 보드 명도 96→90% 미세 허용 검토.*

**시간대 팔레트 4종** (기존 #FF6B9D / #4A90E2 / #FFB347에서 파생)
| 시간대 | body 그라데이션 | 무드색 | 보드 톤 |
|--------|-----------------|--------|---------|
| 아침 | #FFF4EC→#FDE9F1→#EAF3FF | #FFD9A8 | #FFFEFC |
| 점심 | #FFF8E8→#FFEEF5→#E6F4FF | #BFE0FF | #FFFFFF |
| 오후 | #FFF0DC→#FFE4EA→#FFEAD6 | #FFB347 | #FFFCF8 |
| 밤(다크) | #2C2A4A→#3A3060→#46396E | #FF8FB4 | #F6F2FA |

전환은 body·무대만 600ms cross-fade(보드는 색 변화 없음).

**타일**: squircle(radius 30%), 시그니처 radial-gradient. **색약 대비 3중 = 색 + 좌상단 글리프 배지(♥▢~◗▲✦) + 캐릭터.** 5초 무입력 시 매치 가능 타일 breathe 힌트(무압박).

**컷 레이아웃**: 일러(상 64%) + 말풍선 + 하단 내레이션 띠(반투명 .92/밤 .82) + 인디케이터 + 스킵(우상단). 실패 컷은 **경고색 금지**, 톤다운 다독임.

**타이포(Pretendard, 4/8 리듬)**: HUD 라벨 11/700 · 턴 숫자 26/800 · 미션 15/700 · 내레이션 16/600/1.7 · 대사 17/700 · 엔딩 28/900 · 콤보 40/900.

## 7. 모션 규율

- 다이브: `.dive-cam` 단일 노드 줌 + 레이어 패럴랙스(perspective/translateZ), 0.95s 5단계, 풀샷 별도 컷인. 기존 `game.css` 시그니처 이징 `cubic-bezier(.34,1.56,.64,1)` 계승.
- **평소 진폭을 눌러 다이브 대비를 만든다.** 발동 즉시 게이지 0 + 입력잠금, `diveUsedThisRound`로 1회 보장, 라운드 종료 시 `cancel()`.
- 컷: 횡 슬라이드 + 켄번스(느린 scale) + 말풍선 pop.
- **reduced-motion**: 다이브/컷/idle 전부 디졸브·정지로 폴백하되 **정보는 100% 전달**.

## 8. 성능 예산 (실측 기반)

- 목표: 첫 진입 **≤ 300KB** · LCP **≤ 2.0s** · CLS **≤ 0.05**.
- **즉시 수정**: 현 `sw.js`가 install에 icon-512(428KB)까지 프리캐시(총 ~744KB) → **icon-512 install 제외**(fetch 시 lazy 캐시).
- **SW 캐시 3-tier**: `core`(셸: Cache First, install 프리캐시) / `art`(장면 일러: SWR 런타임 캐시 + **LRU 용량 캡**, 무한 증식 방지).
- 큰 일러: **AVIF + webp `<picture>`, 2x 해상도 상한, aspect-ratio 예약(CLS 0)**. 풀샷 디코드 ≈ 6.5MB/장 → **동시 ≤ 3장, 줌 후 노드 즉시 해제**, 사전 `Image.decode()`.
- **다이브 줌 중 backdrop-filter OFF**(블러는 구운 이미지). `will-change`는 다이브 구간 한정 동적 부여/해제.
- 저사양(`hardwareConcurrency ≤ 4` 또는 `deviceMemory ≤ 4`): **라이트 모드**(다이브 → 정적 컷인).
- 측정 체크리스트: Lighthouse 모바일(LCP/CLS/INP), 다이브 중 평균 ≥ 55fps, 메모리 누수, 오프라인 재진입.

## 9. 확정 결정 (team-lead)

1. **다이브 효과** = 방해물 + 목표 세포 일괄 정리(단일, 미션별 분기 ✕). 미션별 차등은 3차로 보류.
2. **soft_fail** = 다음 장면 **해금된다**(클리어 강제 ✕). 단 보상 컷(클리어 4컷·"마을 한 컷")은 클리어 시에만. `cleared`/`visited` 분리 저장.
3. **컷 에셋** = MVP는 챕터 전체 **8컷으로 제한**(24컷 → 8컷). 1장면 풀제작 4컷 + 재사용 4컷. 챕터1 풀세트는 2차(장면당 4컷, 총 ≤ 16컷).
4. **데드락** = `hasPossibleMove()` + 자동 셔플 **1차 필수**(셔플은 벌점 아닌 조용한 연출).

### 정합성 점검 (남은 협의)
- 다이브 "방해물 일괄 제거" ↔ "먹구름 걷어내기" 미션 중복 → 방해물 미션 라운드는 게이지 충전량 ↓로 밸런싱.
- 턴 밸런싱: `moves` 기본값·미션 목표 수치를 `chapter1.json`에 1차부터 명시.
- 밤 다크 ↔ 흰 보드 눈부심 → 보드 명도 미세 조정.

## 10. 마일스톤

**1차 (MVP — 플레이 가능한 1장면)** · 착수 순서
1. 코어 비파괴 리팩터: `SIZE → ROWS/COLS`(6×7), 타이머 제거 → `movesLeft`.
2. `hasPossibleMove()` + 자동 셔플(데드락 차단).
3. SW 3-tier 분리 + icon-512 install 제외(성능 게이트).
4. 상태 머신 + 입력잠금 단일 게이트.
5. mission 모듈 1종(마음 채우기) + engine 이벤트 구독.
6. scene 모듈(정지 컷: 진입 1 + 엔딩 2) + `chapter1.json` 1장면.

✅ 완료 기준: 1장면을 진입컷 → 6×7 미션 클리어/soft_fail → 엔딩컷까지 무빌드 플레이, 데드락 없음, 첫 진입 ≤ 300KB 실측.

**2차 (서사 두께)**: 미션 3종 · 방해물 `tile.kind` · 다이브(게이지+WAAPI 줌) · 챕터1 4장면 · 시간대 동기화 · 진행 저장 · 하루 마감 "마을 한 컷".

**3차 (확장·폴리시)**: 미션별 다이브 차등 · 라이트모드 자동 전환 · art LRU 튜닝 · 챕터2 · 모션 perf 협의 적용 · (선택) 단톡방 공유 카드.
