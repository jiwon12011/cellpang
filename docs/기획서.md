# 세포팡 기획서 v2 — 다중 화면 앱

> "유미의 세포들" IP 모바일 매치-3 **힐링** 퍼즐 PWA.
> 태그라인: **"오늘의 마음, 한 판."**

> 작성: 우리팀 6인 협업 (team-lead 종합). v1은 `docs/_archive/기획서-v1.md`(게임 내부 설계는 계승, 화면 구조를 이번에 신설).

## 0. 비전·포지셔닝

> **"이기는 매치-3가 아니라, 오늘의 감정을 돌보는 매치-3."**
> 점수는 비교를 부르고, **한 컷은 기억을 부른다.** 모이는 건 등수가 아니라 *내가 돌본 하루들의 앨범.*

핵심 전환(이전 불만 해소):
- ❌ 8×8 빽빽 → ✅ **5×7(35칸), 칸 키우고 여백 +50%**
- ❌ 단일 화면 → ✅ **다중 화면 앱**(켜고·고르고·풀고·남기고·돌아오는 흐름)
- ❌ 기획이 화면으로 안 이어짐 → ✅ **화면 맵이 기획의 중심**

## 1. 화면 맵 & 흐름

```
[S1 홈]  시간대 인사 + "마음 지도 열기"
   │
   ▼
[S2 마음 지도]  하루 능선(아침→점심→오후→밤), 클리어=폴라로이드 한 컷
   │ 장면 선택
   ▼
[S3 진입 컷]  유미의 한 장면 (일러+말풍선) → 미션 제시   #scene
   │
   ▼
[S4 플레이]  5×7 보드 한 판 (+ 콤보 다이브)            #play
   │
   ▼
[S5 결과/엔딩 컷]  클리어=한 컷 적립 / "그런 날"=흐린 한 컷   #result
   │
   └──▶ [S2 지도 복귀]   또는   "한 번 더" → S3
```
(2차: S6 한 컷 갤러리 · S7 설정) — 라우팅 해시 `#home/#map/#scene/#play/#result`.

## 2. 화면별 설계

### S1 홈
- 로고/타이틀 블록(세포 마을 일러 + "세포팡" 28/900 + 서브 15/600) → 오늘의 한 컷 카드(361×200, 없으면 플레이스홀더) → 메인 CTA "마음 지도 열기"(361×60, radius 30, 핑크) → 하단 링크(설정/갤러리).
- **시간대 분기 인사**: 첫 방문="유미가 기다렸어요" / 재방문·미완="어제 그 장면, 마저 돌볼까요?" / 시간대 인사(좋은 아침/출출할 시간/오후의 햇살/긴 하루 끝).

### S2 마음 지도 (핵심)
- **하루 능선 = 세로 지그재그 패스**(아침→점심→오후→밤), 배경 = 현재 시간대 무드. 노드 간격 120, 점선 패스.
- **노드 4상태**:
  | 상태 | 형태 | 표현 |
  |------|------|------|
  | 클리어 | 폴라로이드 한 컷 96×120 | 컷 썸네일 + ✓ 금색 배지 |
  | 현재 시간대 | 원형 노드 + 펄스 링 88 | "지금" 라벨, breathe |
  | 방문·미클리어 | 반채움 노드 80 | "다시 해보기" |
  | 잠금 | 흐림 노드 72 | 자물쇠 + "○○시에 열려요" |
- **추천 + 자유**: 현재 시간대를 펄스로 강조하되, 해제된 노드는 자유 선택(시간대가 노드를 잠그지 않음 — 해금 판정은 진행도만).

### S3 진입 컷 / S5 엔딩 컷 (scene.js 공용)
- 일러 영역 상단 64%(켄번스 slow scale) + 말풍선(흰 0.95, 17/700) + 내레이션 띠(하단, 반투명 .92/밤 .82, 16/600) + 인디케이터 + 스킵(우상단 44).
- S3 버튼 "시작" / S5 클리어 "다음 장면"+"한 번 더" / S5 soft_fail "한 번 더"(다독임, **경고색 금지**).

### S4 플레이
- 상단바 56 + **무대 184**(세포 리액션 + 감정 게이지: 숫자 비노출, 그라데이션 바 "감정이 차오른다") + **미션·턴 HUD 52**(좌 미션 칩 / 우 남은 턴) + **보드 카드 496**.
- soft_fail 임박 시 턴 숫자만 핑크로(진동·경고색 금지).

## 3. 보드 & 타일

**보드 5×7 확정 수치**: 타일 **56px** · gap **12** · 카드 패딩 **16** · radius **28** · 화면 마진 **16** → 보드 높이 ≈ 488–496. (8×8 대비 칸+여백 50%↑, 터치 56≥44 ✅)

**타일 = 원형 글로시 버블**: 받침 버블 ⌀50(에셋 캐릭터가 이미 글래스 링 보유 → 동심 정렬), 좌상단 하이라이트 + 입체 그림자 + 유리테. **3중 대비 = 색 + 글리프 배지 + 캐릭터 얼굴**(색약 안전).

## 6. 전환 & 모션

- **깊이 은유**: 전진 = 확대 페이드(공간 안으로), 복귀 = 축소 페이드(밖으로). 슬라이드 남발 ✕.
- **공유 요소**: 지도 카드 ↔ 컷 일러를 transform 보간으로 연결(서사 연속성).
- 전환 base 400ms / 컷 520ms / 복귀 440ms. 동시 렌더 화면 ≤ 2.
- 마이크로: 홈 버튼 breathe, 지도 카드 stagger 등장, 현재 노드 pulse, 말풍선 pop.
- **reduced-motion**: 전부 디졸브·정지로 폴백하되 **정보 100% 전달**. 다이브 → 정적 컷인.

## 7. 상태 & 저장

**구현 방식: 단일 index.html + 해시 라우팅(SPA).** 멀티 HTML은 컷·다이브 연출을 죽이므로 기각.

**모듈**: `app.js`(부트) · `router.js`(해시+전환 라이프사이클) · `appState.js`(저장) · `screens/`(home/map/scene/play/gallery/settings) + v1 코어 7파일(engine/input/mission/gauge…)은 **플레이 전용** 유지(서사·화면 모름).

**2계층 상태 머신**: 화면 레벨(home/map/scene/play/result) ⊃ 플레이 내부(intro→playing→resolved→ending). 입력잠금 = `phase==='playing' && !engine.isResolving`.

**저장 스키마** `localStorage["yumi-cellpang-save"]` (v2):
```json
{
  "v": 2, "currentChapter": "ch1",
  "scenes": { "ch1-morning": { "unlocked": true, "visited": true, "cleared": true } },
  "todayScene": "ch1-morning",
  "collectedCuts": [{ "id": "ch1-morning-clear", "at": "..." }],
  "oneCutOfDay": { "date": "...", "cutId": null },
  "settings": { "reducedMotion": "auto", "sound": true }
}
```
`visited ≠ cleared` (soft_fail은 visited만). v1→v2 마이그레이션 + 파싱 실패 시 안전 리셋.

## 8. 성능 예산 (실측 기반)

- **단일 SPA 확정.** 첫 진입 = 셸 + 홈 1화면(**≤285KB**, 게이트 ≤300KB). S2~S5는 lazy + 다음 화면 prefetch.
- 목표: 홈 LCP ≤ 2.0s · CLS ≤ 0.05 · INP ≤ 200ms · 화면 전환 ≤ 16ms · (다이브 포함) ≥ 55fps.
- **일러 장당 캡** (전부 AVIF+webp · **2x 상한** · aspect-ratio 예약): 지도 썸네일 ≤8KB / 컷 ≤70KB / 풀샷 ≤120KB.
- **SW 3-tier**: core(셸, Cache First, install ≤300KB) / art(일러, SWR, **LRU 5MB**) / runtime. **icon-512 install 제외**.
- 풀샷 동시 디코드 ≤ 1장(줌 후 노드 해제). 저사양(`hardwareConcurrency≤4`/`deviceMemory≤4`/reduced-motion) → 라이트 모드.

## 9. 확정 결정 (team-lead)

1. **플레이 중 새로고침** → 보드 중간상태 저장 ✕, **장면 재시작**(1차 단순성).
2. **플레이 중 뒤로가기** → 1차는 "확인 후 진행 폐기→지도". 더미 history 가드는 2차.
3. **scene.js** → 해시 `#scene`/`#result` **분리**, 컴포넌트는 공용(중복 0).
4. **시간대 자동 vs 수동** → **추천 하이라이트 + 수동 자유 공존**(시간대가 노드를 잠그지 않음).
5. **soft_fail 판정** → 이동 소진 + 목표 미달 = soft_fail. 보드 진척 있으면 **흐린 한 컷 적립**(엔딩컷 CSS 블러 재활용).

**S2 합치 확정**: ideator "하루 능선" = designer "지그재그 패스 + 4상태 노드" — 동일 구조, 단일 화면.

## 11. 마일스톤

**1차 — MVP 한 바퀴(S1→S2→S3→S4→S5→복귀)** · 착수 순서
1. 단일 index.html + router.js(해시 셸)
2. v2 저장 스키마 + 마이그레이션 + 입력잠금 단일 게이트
3. 2계층 상태 머신 골격
4. S4 플레이 코어(v1 7파일 이식, 보드 5×7, 붙는 쌍 회피 생성기, 데드락 셔플)
5. S1 홈(시간대 분기) + S2 지도(지그재그 4상태 노드)
6. scene.js 공용 S3/S5 + oneCutOfDay 적립 + soft_fail 판정
7. 깊이 전환 + 공유 요소 + reduced-motion
8. 디자인 토큰(56/12/16/28/16) + 3중 대비 타일
9. 성능 예산(lazy/prefetch, art LRU, 일러 캡, 라이트 모드)
10. 3장면 콘텐츠 + 한 바퀴 QA

✅ 완료 기준: 홈→지도→진입컷→5×7 클리어/soft_fail→엔딩컷→지도 복귀 무빌드 완주, 데드락 없음, 뒤로가기 정상, 첫 진입 ≤300KB.
